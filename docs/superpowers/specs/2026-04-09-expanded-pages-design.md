## Page 1: Runs (runs.html)

### Data: runs.json

```json
{
  "summary": {
    "our_runs": 45,
    "opp_runs": 38,
    "avg_our_length": 3.8,
    "avg_opp_length": 3.4,
    "longest_our": 8,
    "longest_opp": 6
  },
  "starters": [
    {
      "player": "Name",
      "runs_started": 12,
      "rallies_played": 150,
      "start_rate_pct": 8.0,
      "breakdown": { "kill": 5, "ace": 3, "block": 2, "opp_error": 2 }
    }
  ],
  "killers": [
    {
      "player": "Name",
      "runs_triggered": 8,
      "rallies_played": 150,
      "trigger_rate_pct": 5.3,
      "breakdown": { "attack_error": 4, "srv_error": 2, "other": 2 }
    }
  ],
  "runs_by_phase": [
    { "phase": "early", "our_runs": 15, "opp_runs": 12 }
  ],
  "runs_by_situation": [
    { "situation": "close", "our_runs": 20, "opp_runs": 18 }
  ]
}
```

### Layout

1. **KPI Row (6 cards):** Our Runs, Opp Runs, Avg Our Length, Avg Opp Length, Longest Ours, Longest Theirs
2. **Run Starter Leaderboard:** Sortable table — Player, Runs Started, Rallies Played, Start Rate %, Action Breakdown (kill/ace/block/opp error). Sorted by start rate descending.
3. **Run Killer Leaderboard:** Sortable table — Player, Runs Triggered, Rallies Played, Trigger Rate %, Error Type Breakdown. Sorted by trigger rate descending.
4. **Runs by Game Phase:** Grouped bar chart — Early/Middle/Final, two series (Our Runs, Opp Runs)
5. **Runs by Score Situation:** Grouped bar chart — 5 states, two series (Our Runs, Opp Runs)

### Data Generation

Uses existing `analytics/team.py`:
- `detect_runs(rallies_df)` — returns `{"our_runs": [...], "opp_runs": [...]}`
- `run_triggers(runs, actions_df, our_team_name)` — returns DataFrame of trigger analysis

New computation in `generate_site.py`:
- Rallies played per player: count of rallies where the player has at least one action
- Run start rate: runs_started / rallies_played * 100
- Run killer analysis: for opponent runs, find which of our players' errors triggered them

## Page 2: Game Detail (game.html)

### Data: games.json

```json
{
  "game_list": [
    { "video_id": "abc", "date": "2026-03-15", "opponent": "Club One", "result": "W", "score": "2-0" }
  ],
  "games": {
    "abc": {
      "kpis": { "result": "W", "sets_won": 2, "sets_lost": 0, "sideout_pct": 58.2, "hitting_eff": 0.245, "pass_avg": 2.1 },
      "momentum": [
        { "set_number": 1, "rally_num": 1, "our_score": 1, "opp_score": 0, "score_diff": 1, "point_winner": "us" }
      ],
      "win_probability": [
        { "our_score": 0, "opp_score": 0, "win_pct": 50.0 }
      ],
      "box_scores": {
        "1": [
          { "player": "Name", "kills": 5, "errors": 1, "attempts": 12, "hitting_eff": 0.333, "aces": 1, "digs": 3 }
        ]
      }
    }
  }
}
```

### Layout

1. **Game Selector:** Dropdown showing "Date — Opponent (W/L score)"
2. **Match Summary KPIs (5 cards):** Result badge, Set Score, Sideout %, Hitting Eff, Pass Avg
3. **Momentum Chart:** Plotly line chart per set. X = rally number, Y = score differential. Runs visualized using Plotly shape annotations: green semi-transparent rectangles behind our 3+ point runs, red behind opponent runs. Sets shown as stacked subplots (one chart per set, vertically arranged).
4. **Win Probability Curve:** Plotly line chart per set. X = rally number, Y = win probability %. Overlaid on or adjacent to momentum chart.
5. **Set-by-Set Box Score:** One sortable table per set. Columns: Player, Kills, Errors, Attempts, Hitting Eff, Aces, Digs. Tab or accordion to switch sets.

### Data Generation

Uses existing `analytics/advanced.py`:
- `momentum_data(rallies_df, video_id)` — per-rally score differential
- `win_probability_table(rallies_df)` — historical win % from each score state

New computation in `generate_site.py`:
- Per-game KPIs (already partially exists in overview generation, extract to shared helper)
- Per-set box scores: group actions by video_id + set_number, compute player stats per set
- Run detection per game for momentum chart shading

## Page 3: Zones (zones.html)

### Data: zones.json

```json
{
  "attack_zones": {
    "1": { "eff": 0.15, "kills": 10, "errors": 3, "attempts": 50, "top_player": "Name" },
    "2": { "eff": 0.25, "kills": 20, "errors": 5, "attempts": 60, "top_player": "Name" }
  },
  "receive_zones": {
    "1": { "avg": 2.1, "total": 30, "top_player": "Name" },
    "5": { "avg": 1.8, "total": 45, "top_player": "Name" }
  },
  "attack_detail": [
    { "zone": 1, "kills": 10, "errors": 3, "attempts": 50, "eff": 0.140, "top_player": "Name" }
  ],
  "receive_detail": [
    { "zone": 1, "avg": 2.1, "total": 30, "top_player": "Name" }
  ],
  "player_list": ["Player A", "Player B"],
  "per_player_attack": {
    "Player A": { "1": { "eff": 0.2, "kills": 5, "attempts": 20 } }
  },
  "per_player_receive": {
    "Player A": { "5": { "avg": 2.0, "total": 15 } }
  }
}
```

### Layout

1. **Player Filter:** Dropdown — "All Players" (default) or a specific player. Changes both courts and tables.
2. **Attack Court Heatmap:** SVG volleyball court (zones 1-6) colored by hitting efficiency. Color scale: red (low) → yellow → green (high). Hover tooltip: Zone, Eff, Kills/Attempts, Top Player.
3. **Attack Zone Detail Table:** Sortable table — Zone, Kills, Errors, Attempts, Eff, Top Player.
4. **Receive Court Heatmap:** Same SVG court colored by pass average (0-3). Color scale: red (0) → green (3).
5. **Receive Zone Detail Table:** Sortable table — Zone, Pass Avg, Total Passes, Top Player.

### Court SVG

Standard volleyball court zones (matching `components/court.py` ZONE_COORDS):
- Zone 1: back right
- Zone 2: front right
- Zone 3: front center
- Zone 4: front left
- Zone 5: back left
- Zone 6: back center

Rendered as SVG rectangles. The JS draws the court — no external image needed.

### Data Generation

Uses existing `pages/zones.py` logic (recomputed in `generate_site.py`):
- Group attack actions by `src_zone`, compute kills/errors/attempts/eff per zone
- Group receive actions by `src_zone`, compute pass avg per zone
- Top player per zone (most kills for attack, most passes for receive)
- Per-player versions of the same for the player filter

## Enhancements to Existing Pages

### Overview — Expected Sideout by Pass Quality

**New section** added after Sideout by Phase.

**Container ID:** `expected-sideout-chart`

**Chart:** Bar chart. X = Pass Quality (0, 1, 2, 3). Y = Sideout %. Bars colored on a gradient from red (0) to green (3). Hover shows rally count.

```json
  { "pass_quality": 3, "sideout_pct": 68.0, "rallies": 120 }
]
```

Uses `analytics/advanced.py:expected_sideout_by_pass()`.

### Players — Serve Pressure Index

**New section** added after Consistency Index.

**Container ID:** `serve-pressure`

**Layout:** 5 KPI cards: Total Serves, Aces, Serve Errors, Pressure Serves, Pressure %. Pressure = aces + serves causing opponent 0/1 pass.

**Data:** Added to `players.json` as `serve_pressure` key:
```json
"serve_pressure": {
  "Player A": { "serves": 80, "aces": 12, "srv_errors": 5, "pressure_serves": 28, "pressure_pct": 35.0 }
}
```

Uses `analytics/advanced.py:serve_pressure_index()`.

### Players — In-System vs Out-of-System Efficiency

**New section** added after Serve Pressure.

**Container ID:** `in-system-eff`

**Layout:** Two side-by-side bars: In-System Hitting Eff vs Out-of-System Hitting Eff. Plus text showing attempts for each.

**Data:** Added to `players.json` as `in_system` key:
